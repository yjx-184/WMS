# 库存查询与流水说明

## 库存查询

**端点**：`GET /api/v1/inventory`

支持以下查询条件：

| 参数 | 说明 |
|------|------|
| `product_id` | 按商品筛选 |
| `warehouse_id` | 按仓库筛选 |
| `location_id` | 按库位筛选 |
| `keyword` | 关键字搜索（商品名称/SKU/仓库名称/库位编码） |
| `page` / `page_size` | 分页 |

**返回字段**：商品名称、SKU、仓库名称、库位编码、库存数量、更新时间。

**前端页面**：`/inventory`（`InventoryQuery.tsx`），只读表格，无操作列。

## 库存流水查询

**端点**：`GET /api/v1/inventory-transactions`

**定位**：后端/审计使用，**无前端页面**。

支持筛选条件：`product_id`、`warehouse_id`、`location_id`、`change_type`（inbound/outbound）、日期范围、分页。

返回字段含 `quantity`（变更量）、`quantity_before`（变更前）、`quantity_after`（变更后）、`reference_type`（关联业务类型）、`reference_id`（关联业务 ID）。

## 库存更新的两种方式

### 入库增加：UPSERT

```sql
INSERT INTO inventories (product_id, warehouse_id, location_id, quantity)
VALUES ($1, $2, $3, $4)
ON CONFLICT (product_id, warehouse_id, location_id)
DO UPDATE SET quantity = inventories.quantity + $4
RETURNING *
```

- 如果 `(商品, 仓库, 库位)` 组合**不存在**：插入新行，quantity = delta
- 如果**已存在**：原子累加 quantity = quantity + delta
- `RETURNING *` 返回更新后的完整行

### 出库扣减：乐观锁

```sql
UPDATE inventories
SET quantity = quantity - $4
WHERE product_id = $1 AND warehouse_id = $2 AND location_id = $3
  AND quantity >= $4
RETURNING *
```

- `WHERE quantity >= $4` 确保库存充足
- 如果库存不足 → 更新 0 行 → 返回 None → Service 层报错
- 不需要显式锁定，利用 SQL 的原子性

## quantity_before / quantity_after 的计算

**不额外查询**当前库存，而是**从原子操作的返回值反推**：

```
增加：before = after - delta
减少：before = after + delta
```

例如：
- 入库 100 个，upsert 返回 quantity=150 → before=150-100=50, after=150
- 出库 30 个，decrease 返回 quantity=70 → before=70+30=100, after=70

这样做的好处是 `before`/`after` 与原子操作完全一致，避免了"先查询再修改"之间的竞态问题。
