# 前端架构与页面路由

## 技术栈

| 层面 | 技术 |
|------|------|
| UI 框架 | React 18 |
| 构建工具 | Vite 5 |
| UI 组件库 | Ant Design 5 |
| 路由 | React Router 6 |
| HTTP 客户端 | Axios（统一拦截器处理错误） |
| 服务端状态 | TanStack Query（自动缓存、重新获取） |

## 页面路由表

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | → `/products` | 默认跳转 |
| `/products` | ProductList | 商品列表 |
| `/warehouses` | WarehouseList | 仓库列表 |
| `/warehouses/:id/locations` | LocationList | 库位管理 |
| `/inbounds` | InboundList | 入库单列表 |
| `/inbounds/new` | InboundForm | 新增入库单 |
| `/inbounds/:id` | InboundDetail | 入库单详情 |
| `/outbounds` | OutboundList | 出库单列表 |
| `/outbounds/new` | OutboundForm | 新增出库单 |
| `/outbounds/:id` | OutboundDetail | 出库单详情 |
| `/outbounds/:id/edit` | OutboundForm(编辑) | 编辑出库单 |
| `/inventory` | InventoryQuery | 库存查询 |
| `*` | NotFound | 404 |

## 数据流

```
用户操作 → React 状态变更 → TanStack Query 自动请求 API
                                    ↓
                              apiClient (Axios)
                                    ↓
                        后端 /api/v1/* 端点
                                    ↓
                        Axios 响应拦截器检查 code
                                    ↓
                    code===0 → 返回 data
                    code!==0 → message.error() 弹窗
```

## Axios 客户端封装

`frontend/src/api/client.ts`：

- `baseURL: '/api/v1'` — 所有 API 请求自动加前缀
- `timeout: 10000` — 10 秒超时
- 响应拦截器：自动检查 `code` 字段，非 0 时弹出错误提示

所有业务 API 模块（product.ts, inbound.ts 等）都复用这个 client，不会单独创建 Axios 实例。

## 状态管理

使用 **TanStack Query**（原 React Query）管理服务端状态：

- `useQuery`：获取数据（自动缓存、后台刷新）
- `useMutation`：修改数据（创建、更新、删除）
- `queryClient.invalidateQueries`：数据变更后刷新相关缓存

不需要 Redux/Zustand 等额外的状态管理库。
