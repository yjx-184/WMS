# 本地运行与测试指南

## 环境要求

- **Rust** 1.80+
- **Node.js** 20+
- **Docker Desktop**（用于运行 PostgreSQL）

## 第一步：启动数据库

```bash
docker compose up -d
```

这会启动 PostgreSQL 16，端口 5432，数据库 wms，用户名/密码 wms/wms123。

## 第二步：启动后端

```bash
cd backend
cargo run
```

后端启动时会自动：
1. 执行数据库迁移（建表）
2. 检查数据库是否为空 → 自动写入种子数据（5 个商品、1 个仓库、4 个库位）
3. 监听 `http://localhost:3000`

验证：`curl http://localhost:3000/api/v1/health` 返回 `{"code":0}`

### 手动执行种子数据

```bash
cargo run -- --seed
```

种子数据是幂等的，重复执行不会产生重复记录。

## 第三步：启动前端

```bash
cd frontend
npm install
npm run dev
```

前端监听 `http://localhost:5173`。后端已配置 CORS 允许该来源的跨域请求，
前端 Axios 的 `baseURL` 为 `/api/v1`，需要配合 Vite proxy 或
直接让 Axios 请求完整后端地址才能正常调用 API。

**推荐配置 Vite proxy**（`vite.config.ts`）：

```ts
export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': 'http://localhost:3000',
    },
  },
});
```

若未配置 proxy，前端 API 请求会因跨域或路径问题无法到达后端。

## 运行测试

```bash
cd backend

# 首先启动测试数据库
docker compose up -d postgres_test

# 运行所有测试
cargo test
```

测试数据库在端口 5433，数据库名 `wms_test`，与开发数据库完全隔离。

测试会在每次运行时自动执行迁移并清理测试数据。

## 前端构建

```bash
cd frontend
npm run build
```

构建产物在 `frontend/dist/`。

## 项目目录结构速查

| 目录/文件 | 说明 |
|-----------|------|
| `backend/src/main.rs` | 后端入口 |
| `backend/src/router.rs` | 全部 API 路由注册 |
| `backend/migrations/` | 数据库建表 SQL |
| `backend/migrations/seed.sql` | 种子数据 SQL |
| `backend/tests/` | 集成测试 |
| `frontend/src/pages/` | 页面组件 |
| `frontend/src/api/client.ts` | Axios 封装 |
| `frontend/src/router/index.tsx` | 前端路由表 |
| `docker-compose.yml` | 数据库容器定义 |
| `docs/` | 文档（你正在看这里） |

## 成功演示流程

1. 启动数据库 `docker compose up -d`
2. 启动后端 `cd backend && cargo run`
3. 配置 Vite proxy（见上方第三步说明）
4. 启动前端 `cd frontend && npm run dev`
5. 浏览器打开 `http://localhost:5173`
3. 看到 5 个种子商品
4. 创建入库单 → 完成入库 → 库存增加
5. 创建出库单 → 完成出库 → 库存减少
6. 库存查询页验证最终结果
