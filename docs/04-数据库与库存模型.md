# 数据库与库存模型

## 数据库概览

使用 **PostgreSQL 16**，共 **9 张业务表**。

## 表结构

### 主数据表（3 张）

| 表 | 说明 | 关键字段 |
|----|------|----------|
| `products` | 商品 | `sku_code`(唯一), `name`, `unit`, `status`(active/disabled) |
| `warehouses` | 仓库 | `code`(唯一), `name`, `address` |
| `locations` | 库位 | `code`, `location_type`(normal/receiving/shipping/return), 同仓库内 code 唯一 |

### 库存表（1 张）

| 表 | 说明 | 唯一约束 |
|----|------|----------|
| `inventories` | 库存快照 | `(product_id, warehouse_id, location_id)` — 三元组唯一 |

库存数量按**商品 + 仓库 + 库位**三个维度记录。同一商品在不同库位有独立的库存数量。

### 入库表（2 张）

| 表 | 说明 |
|----|------|
| `inbound_orders` | 入库单主表（单号、仓库、状态） |
| `inbound_order_items` | 入库单明细（商品、库位、计划数量、实收数量） |

### 出库表（2 张）

| 表 | 说明 |
|----|------|
| `outbound_orders` | 出库单主表 |
| `outbound_order_items` | 出库单明细 |

### 流水表（1 张）

| 表 | 说明 |
|----|------|
| `inventory_transactions` | 库存流水（每次库存变更记录一条） |

## ENUM 类型

| ENUM | 值 | 说明 |
|------|-----|------|
| `product_status` | active / disabled | 商品状态 |
| `warehouse_status` | active / disabled | 仓库状态 |
| `location_type` | normal / receiving / shipping / return | 库位类型 |
| `inbound_order_type` | purchase / return / manual | 入库类型 |
| `inbound_order_status` | **draft / completed / cancelled** | 入库单状态 |
| `outbound_order_type` | sales / manual / scrap | 出库类型 |
| `outbound_order_status` | **draft / completed / cancelled** | 出库单状态 |
| `transaction_change_type` | **inbound / outbound** | 流水变更类型 |

## 库存三元组

库存的粒度是 `(product_id, warehouse_id, location_id)` 三元组：

```
商品 iPhone 15 → 主仓库 → A-01-01 库位 → 100 个
商品 iPhone 15 → 主仓库 → A-01-02 库位 → 50 个
```

同一个商品在同一个仓库的不同库位，库存是独立记录的。

## 库存流水表

每次库存变更自动写入一条流水记录：

| 字段 | 说明 |
|------|------|
| `change_type` | inbound（入库增加）或 outbound（出库减少/回滚） |
| `quantity` | 本次变更数量 |
| `quantity_before` | 变更前的库存数量 |
| `quantity_after` | 变更后的库存数量 |
| `reference_type` | 关联业务类型（如 `inbound_order_complete`） |
| `reference_id` | 关联业务 ID（如入库单 ID） |

通过流水表可以追溯任意时刻的库存变化历史。
